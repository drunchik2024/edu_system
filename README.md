# ИС Управления образовательными программами — ВГТУ

Веб-приложение для управления образовательными программами кафедры: ведение программ, дисциплин, чек-листов и отчётов.

## Стек

- **Backend**: Python 3.11+ · FastAPI · SQLAlchemy 2 (async) · Alembic
- **БД**: PostgreSQL 14+
- **Frontend**: Jinja2 · Tailwind CSS (CDN) · Vanilla JS
- **Аутентификация**: JWT (cookie, 8 часов)

---

## Роли пользователей

| Роль | Код роли | Возможности |
|------|----------|-------------|
| Заведующий кафедрой | `head_of_department` | Просмотр программ и дисциплин кафедры; управление чек-листами и отчётами; утверждение отчётов |
| Руководитель ОП (РОП) | `program_director` | Создание и редактирование своих образовательных программ; загрузка PDF-документов и рецензий; управление компетенциями |
| Преподаватель | `teacher` | Просмотр программ и дисциплин; подача отчётов |
| Декан | `dean` | Просмотр (аналогично заведующему) |

## Функциональные модули

### Образовательные программы (`/programs`)

- Список программ кафедры (для заведующего/преподавателя) или своих программ (для РОП)
- Детальная карточка программы с дисциплинами своей кафедры и внешними дисциплинами
- Форма создания и редактирования программы (уровень, форма обучения, срок, описание, виды деятельности, структура, практики)

### Модуль РОП (`/programs/new`, `/programs/{id}/edit`)

Доступен только пользователям с ролью `program_director`:

- Создание / редактирование / удаление образовательной программы
- Загрузка PDF-документов: стандарт (ФГОС), учебный план, титульный лист
- Рецензии — загрузка PDF-файлов с наименованием рецензента
- Компетенции — привязка УК / ОПК / ПК из общего справочника; создание новых компетенций прямо из формы
- Список дисциплин программы (текстовый)
- Области профессиональной деятельности (номер + описание)

### Дисциплины (`/disciplines`)

- Список дисциплин кафедры с фильтрацией по программе и сортировкой по названию / шифру
- Карточка дисциплины: цели, задачи, результаты, преподаватели

### Отчёты / чек-листы (`/reports/discipline/{id}`)

- Чек-лист по дисциплине, разбитый по категориям (РПД, ФОС и др.)
- Добавление / удаление категорий и пунктов (заведующий кафедрой)
- Отметка пунктов выполненными, добавление комментариев
- Прикрепление файлов к категориям; обратная связь заведующего по файлам
- Жизненный цикл отчёта: `черновик → на утверждении → утверждён`
- Заведующий кафедрой утверждает отчёт сразу при подаче (без промежуточного статуса)
- Скачивание прикреплённых файлов

## Быстрый старт

### 1. Виртуальное окружение и зависимости

```bash
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv\Scripts\activate         # Windows

pip install -r requirements.txt
```

### 2. База данных и `.env`

```bash
# Создать БД в PostgreSQL
psql -U postgres -c "CREATE DATABASE edu_system;"

# Скопировать и заполнить .env
cp .env.example .env
# Отредактировать DATABASE_URL и SECRET_KEY
```

Пример `.env`:
```
DATABASE_URL=postgresql+asyncpg://postgres:password@localhost:5432/edu_system
SECRET_KEY=your-secret-key-change-in-production-min-32-chars
ACCESS_TOKEN_EXPIRE_MINUTES=480
```

### 3. Миграции

```bash
alembic upgrade head
```

### 4. Тестовые данные

```bash
python seed.py
```

Скрипт пересоздаёт схему и наполняет БД: 4 кафедры, 10 образовательных программ, 24 дисциплины кафедры ИВТ + 8 внешних, справочник компетенций, демо-данные для модуля РОП.

### 5. Запуск

```bash
python main.py
# или
uvicorn app.main:app --reload
```

Открыть в браузере: http://localhost:8000

## Управление миграциями

```bash
# Создать миграцию после изменения моделей
alembic revision --autogenerate -m "описание изменения"
alembic upgrade head
```

## Добавление новой роли

Роли хранятся в таблице `roles` и не требуют изменения кода:

```sql
INSERT INTO roles (name, display_name) VALUES ('new_role', 'Новая роль');
```

В нужном роутере использовать зависимость `require_role("new_role")` из `app/auth.py`.
